Decide which model categories (open-weights, hosted-API, on-prem), hosting regions, and integration patterns are acceptable for the use case. Document why and against which criteria - sovereignty, data sensitivity, performance, cost, sustainability.

Record an exit strategy for every external dependency: how would the assistant continue to work if this supplier became unavailable or unsuitable?

Prefer EU-hosted and open components where the use case allows; document explicitly where deviation is accepted.
